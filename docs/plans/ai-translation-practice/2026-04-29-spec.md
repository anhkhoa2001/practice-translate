# Spec: AI Translation Practice (Angular Frontend)

## Ngày: 2026-04-29
## Trạng thái: Đã duyệt

---

## 1. Tổng quan

Ứng dụng luyện dịch tiếng Anh sang tiếng Việt, chạy hoàn toàn trên Frontend (Angular 17+). Ngườ dùng tự nhập Gemini API Key. App tách văn bản thành các câu, highlight từng câu để ngườ dùng dịch, sau đó gọi AI để nhận xét bản dịch.

---

## 2. Yêu cầu chức năng (MVP)

### 2.1 Nhập API Key
- Màn hình đầu tiên yêu cầu nhập Gemini API Key
- Lưu vào `localStorage`
- Cho phép thay đổi key sau này

### 2.2 Dán văn bản tiếng Anh
- Textarea để ngườ dùng dán đoạn văn
- Nút "Bắt đầu luyện dịch" để tách văn bản

### 2.3 Tách câu
- Sử dụng `Intl.Segmenter` (browser API) để tách văn bản thành mảng câu
- Fallback: tách theo dấu chấm nếu trình duyệt không hỗ trợ

### 2.4 Giao diện luyện dịch (Layout 2 cột)
- **Cột trái**: Danh sách các câu tiếng Anh. Câu đang làm việc được highlight (ví dụ: nền vàng hoặc viền xanh)
- **Cột phải**: 
  - Ô nhập bản dịch tiếng Việt
  - Nút "Gửi" (Submit) để gọi AI nhận xét
  - Nút "Gợi ý" để lấy keywords từ AI
  - Panel hiển thị feedback từ AI

### 2.5 Phím tắt
- `Enter`: Submit bản dịch hiện tại
- `↑/↓`: Di chuyển giữa các câu

### 2.6 Feedback AI
- Sau khi submit, gọi Gemini API với prompt nhận xét bản dịch
- Response JSON chứa danh sách nhận xét, mỗi nhận xét có:
  - `type`: `"good"` | `"error"`
  - `message`: Nội dung nhận xét
- Hiển thị dưới dạng **thẻ màu**:
  - 🟢 Xanh: Chỗ dịch tốt
  - 🔴 Đỏ: Lỗi sai cần sửa

### 2.7 Luồng chuyển câu
- Sau khi nhận được feedback, hiển thị nút **"Tiếp theo"**
- Ngườ dùng chủ động nhấn nút để sang câu kế tiếp
- Cho phép sửa lại bản dịch và submit lại nếu muốn

### 2.8 Gợi ý Keywords
- Nút "Gợi ý" gọi API riêng để lấy các từ khóa quan trọng của câu đang dịch
- Hiển thị gợi ý ngay dưới ô nhập

## 3. Kiến trúc kỹ thuật

### 3.1 Công nghệ
- **Framework**: Angular 17+ (Standalone Components)
- **State Management**: Angular Signals (`WritableSignal`)
- **HTTP Client**: Angular `HttpClient` gọi trực tiếp Gemini REST API
- **Styling**: Angular Material hoặc CSS/Tailwind (tuỳ chọn)

### 3.2 Cấu trúc thư mục đề xuất
```
src/app/
  components/
    api-key-setup/
    reader/
    sentence-card/
    feedback-panel/
    hint-button/
  services/
    gemini.service.ts
    text-processor.service.ts
    storage.service.ts
  models/
    sentence.model.ts
    feedback.model.ts
  app.component.ts
```

### 3.3 Services

#### `StorageService`
- Đọc/ghi API key từ `localStorage`
- Key: `gemini_api_key`

#### `TextProcessorService`
- Input: `string` (đoạn văn tiếng Anh)
- Output: `Sentence[]`
- Tách câu bằng `Intl.Segmenter` với `granularity: 'sentence'`

#### `GeminiService`
- Input: `(original: string, translation: string)`
- Output: `Feedback[]`
- Xây dựng prompt để nhận xét bản dịch
- Parse JSON response
- Method riêng cho gợi ý keywords: `getKeywords(original: string)`

### 3.4 Data Models

```typescript
interface Sentence {
  id: number;
  original: string;
  translation: string;
  feedback: Feedback[];
  keywords: string[];
}

interface Feedback {
  type: 'good' | 'error';
  message: string;
}
```

### 3.5 State (Signals)

| Signal | Type | Mô tả |
|--------|------|-------|
| `apiKey` | `WritableSignal<string>` | API key hiện tại |
| `sentences` | `WritableSignal<Sentence[]>` | Danh sách câu |
| `currentIndex` | `WritableSignal<number>` | Index câu đang chọn |
| `isLoading` | `WritableSignal<boolean>` | Trạng thái chờ API |
| `currentFeedback` | `WritableSignal<Feedback[]>` | Feedback của câu hiện tại |

## 4. Xử lý lỗi

| Lỗi | Cách xử lý |
|-----|-----------|
| API key không hợp lệ / hết quota | Hiển thị thông báo lỗi rõ ràng, cho phép nhập lại key |
| Lỗi mạng | Hiển thị nút "Thử lại" |
| Response AI không đúng JSON | Fallback: hiển thị text thô từ AI |
| Văn bản quá dài | Giới hạn độ dài đầu vào (ví dụ: 5000 ký tự) |

## 5. Prompt thiết kế (Gemini)

### 5.1 Prompt nhận xét bản dịch
```
Bạn là một giáo viên dạy dịch thuật tiếng Anh sang tiếng Việt.
Hãy nhận xét bản dịch sau của học viên.

Câu gốc tiếng Anh: "{original}"
Bản dịch tiếng Việt: "{translation}"

Trả lờ dưới dạng JSON với định dạng:
[
  {"type": "good", "message": "..."},
  {"type": "error", "message": "..."}
]

- type="good": Chỗ dịch đúng, tự nhiên
- type="error": Lỗi sai về ngữ pháp, từ vựng, hoặc diễn đạt không tự nhiên
```

### 5.2 Prompt gợi ý keywords
```
Cho câu tiếng Anh sau: "{original}"
Hãy đưa ra 3-5 từ khóa quan trọng cần chú ý khi dịch sang tiếng Việt.
Trả lờ dạng JSON: {"keywords": ["từ 1", "từ 2", ...]}
```

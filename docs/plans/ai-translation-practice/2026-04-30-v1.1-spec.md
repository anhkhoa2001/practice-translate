# Spec: AI Translation Practice v1.1

## Ngày: 2026-04-30
## Trạng thái: Đã duyệt

---

## 1. Tổng quan

Version 1.1 mở rộng app luyện dịch với 4 cải tiến chính: tăng giới hạn text + thêm tiêu đề, lưu session vào GitHub repo cố định, thêm nhập GitHub PAT ở màn hình setup, và mở rộng cache localStorage cho cả 2 key.

---

## 2. Thay đổi từ v1.0

### 2.1 Mở rộng màn hình nhập đoạn văn
- Tăng `maxTextLength` từ 5000 → 10000 ký tự
- Thêm input "Tiêu đề bài viết" (bắt buộc) ở trên textarea nhập đoạn văn
- Tiêu đề hiển thị ở reader header khi đang dịch
- Tiêu đề dùng làm phần tên file khi lưu vào GitHub (slugified)

### 2.2 Lưu đoạn văn vào GitHub repo cố định
- Khi nhấn "Bắt đầu luyện dịch", app gọi GitHub Contents API để tạo file JSON trong repo cố định
- File JSON format: `sessions/{timestamp}-{slug}.json`
- Nội dung JSON: `{ title, originalText, sentences, createdAt }`
- Repo name hardcode trong code (ví dụ: `{owner}/ai-translate-sessions`)
- UX: hiển thị trạng thái lưu nhỏ ở header ("Đang lưu..." / "Đã lưu ✓" / "Lỗi lưu")

### 2.3 Màn hình nhập thêm GitHub PAT
- `ApiKeySetupComponent` thêm input thứ 2: "GitHub Personal Access Token"
- Cả Gemini API key và GitHub PAT đều required
- GitHub PAT cần scope `repo` để commit file vào repo
- Thêm link hướng dẫn tạo PAT trong UI

### 2.4 Cache API key (mở rộng)
- `StorageService` thêm 3 methods: `getGithubToken()`, `setGithubToken()`, `clearGithubToken()`
- localStorage keys: `gemini_api_key` (đã có), `github_token` (mới)
- AppComponent kiểm tra cả 2 key khi khởi động
- Nếu thiếu 1 trong 2 → hiển thị màn hình setup

## 3. Kiến trúc thay đổi

### 3.1 Service mới: `GithubService`
- `saveSession(title: string, originalText: string, sentences: Sentence[]): Observable<GithubSaveResult>`
- Gọi `PUT https://api.github.com/repos/{owner}/{repo}/contents/sessions/{filename}`
- Base64 encode JSON content
- Return `{ success: boolean, url?: string, error?: string }`

### 3.2 Data mới: `ArticleSession`
```typescript
interface ArticleSession {
  title: string;
  originalText: string;
  sentences: Sentence[];
  createdAt: string; // ISO 8601
}
```

### 3.3 Thay đổi `StorageService`
- Thêm methods mới cho GitHub token
- Giữ nguyên logic Gemini key

### 3.4 Thay đổi `ApiKeySetupComponent`
- Thêm GitHub PAT input
- Emit cả 2 key khi submit
- Validate cả 2 fields không blank

### 3.5 Thay đổi `AppComponent`
- Kiểm tra cả `gemini_api_key` và `github_token`
- Pass GitHub token vào `GithubService`

### 3.6 Thay đổi `ReaderComponent`
- Thêm `title` field
- Tăng `maxTextLength` lên 10000
- Gọi `GithubService.saveSession()` khi startPractice
- Hiển thị trạng thái lưu

## 4. Xử lý lỗi mới

| Lỗi | Cách xử lý |
|-----|-----------|
| GitHub PAT không hợp lệ | Thông báo lỗi, cho nhập lại key |
| Repo không tồn tại | Thông báo "Repo chưa tồn tại. Vui lòng tạo repo trước." |
| File đã tồn tại (trùng tên) | Tự động thêm suffix timestamp để tránh conflict |
| Lỗi mạng khi lưu GitHub | Hiển thị "Lỗi lưu" nhưng không block UX, vẫn cho dịch |

# 1. Context

**Problem:**
Xây dựng ứng dụng luyện dịch tiếng Anh sang tiếng Việt chạy hoàn toàn trên Frontend. Ngườ dùng tự nhập Gemini API Key, app tách văn bản thành các câu, highlight từng câu để dịch, sau đó gọi AI để nhận xét bản dịch dưới dạng thẻ màu.

**Affected Modules:**
- `src/app/components/` — UI components (api-key-setup, reader, sentence-card, feedback-panel, hint-button)
- `src/app/services/` — business logic (gemini, text-processor, storage)
- `src/app/models/` — data contracts (Sentence, Feedback interfaces)

**Non-Goals:**
- Không có backend/server-side processing
- Không lưu trữ lâu dài ("Sổ tay cá nhân" để dành version sau)
- Không có authentication hay user management
- Không hỗ trợ offline mode ngoài việc lưu API key

---

# 2. Constraints

**Language:** TypeScript
**Architecture:** Component-based (Angular Standalone), Signal-driven state management

**Rules:**
- Do NOT modify unrelated modules
- Do NOT introduce new external dependencies (ngoài Angular CLI defaults)
- Sử dụng `HttpClient` có sẵn của Angular để gọi Gemini REST API
- Tất cả components phải là Standalone Components (Angular 17+)

**Performance Budgets:**
- Latency gọi Gemini API: < 5s (phụ thuộc mạng và model)
- Tách câu (local): < 100ms cho 5000 ký tự
- UI render: < 50ms sau khi sentences signal thay đổi

# 3. Conventions

> Shared rules ALL tasks must follow. Prevents inconsistency when tasks are implemented independently.

**Naming:**
- Components: `PascalCase` + suffix `Component` — e.g. `ReaderComponent`, `SentenceCardComponent`
- Services: `PascalCase` + suffix `Service` — e.g. `GeminiService`, `StorageService`
- Methods/variables: `camelCase` — e.g. `splitSentences()`, `currentIndex`
- Constants: `UPPER_SNAKE_CASE` — e.g. `MAX_TEXT_LENGTH`
- Files: `kebab-case` — e.g. `api-key-setup.component.ts`

**Error Handling:**
- Strategy: return `Result<T, E>` pattern hoặc throw domain-specific errors
- Pattern: `throw new GeminiApiError("Invalid API key")`
- Never return null for arrays — return empty array `[]`
- Validation: fail-fast at service method boundaries

**Logging:**
- Không dùng thư viện logging ngoài — dùng `console.error` cho lỗi, `console.warn` cho cảnh báo
- Required tại: error/catch blocks trong services

**Testing:**
- Framework: Jasmine + Karma (Angular CLI default)
- Naming: `should <expected> when <condition>`
- Example: `should return sentences array when text has multiple sentences`
- Coverage: >= 80% line coverage on new service code

**Task Size:**
- Max 200 LOC per task (excluding tests)
- If a task exceeds this, split into subtasks

# 4. Contracts

> Define ALL interfaces and data structures BEFORE task specs.
> Tasks reference these contracts — not the other way around.

## Interface: `StorageService`

```
StorageService.getApiKey(): string | null
  - pre: none
  - post: returns API key from localStorage, or null if not set
  - throws: none

StorageService.setApiKey(key: string): void
  - pre: key is non-blank
  - post: key is persisted to localStorage under "gemini_api_key"
  - throws: none (trims blank input)

StorageService.clearApiKey(): void
  - pre: none
  - post: removes "gemini_api_key" from localStorage
  - throws: none
```

## Interface: `TextProcessorService`

```
TextProcessorService.splitToSentences(text: string): Sentence[]
  - pre: text is non-blank
  - post: returns array of Sentence objects with sequential ids starting from 0
  - throws: none (returns empty array for blank/empty input)
  - note: uses Intl.Segmenter with granularity 'sentence'; falls back to regex split by [\.!?] + whitespace
```

## Interface: `GeminiService`

```
GeminiService.evaluateTranslation(original: string, translation: string): Promise<Feedback[]>
  - pre: apiKey is set (non-blank); original and translation are non-blank
  - post: returns parsed Feedback array from Gemini API response
  - throws: GeminiApiError when API returns error (invalid key, rate limit, etc.)
  - throws: ParseError when response is not valid JSON

GeminiService.getKeywords(original: string): Promise<string[]>
  - pre: apiKey is set (non-blank); original is non-blank
  - post: returns array of keyword strings from Gemini API response
  - throws: GeminiApiError when API returns error
  - throws: ParseError when response is not valid JSON
```

---

## Data Structure: `Sentence`

```
Sentence {
  id:        number         — sequential index, 0-based
  original:  string         — non-blank, câu tiếng Anh gốc
  translation: string       — bản dịch tiếng Việt của ngườ dùng (có thể blank)
  feedback:  Feedback[]     — mặc định [], cập nhật sau khi submit
  keywords:  string[]       — mặc định [], cập nhật sau khi nhấn "Gợi ý"
}
```

## Data Structure: `Feedback`

```
Feedback {
  type:    'good' | 'error'  — required, không có giá trị khác
  message: string            — non-blank, nội dung nhận xét từ AI
}
```

## Data Structure: `AppState` (Signals)

```
AppState {
  apiKey: WritableSignal<string>          — lưu trữ API key hiện tại
  sentences: WritableSignal<Sentence[]>   — danh sách câu đã tách
  currentIndex: WritableSignal<number>    — index câu đang làm việc, mặc định 0
  isLoading: WritableSignal<boolean>      — true khi đang gọi Gemini API
  currentFeedback: WritableSignal<Feedback[]>  — feedback của câu hiện tại
}
```

# 5. Target Architecture

**Components:**
- `ApiKeySetupComponent` — form nhập và lưu API key vào localStorage
- `ReaderComponent` — layout chính 2 cột, quản lý state và điều phối
- `SentenceCardComponent` — hiển thị 1 câu gốc + ô nhập bản dịch + highlight khi active
- `FeedbackPanelComponent` — render thẻ màu Xanh/Đỏ từ feedback array
- `HintButtonComponent` — nút gọi API lấy keywords, hiển thị kết quả

**Interaction Flow:**

```
User opens app
  → AppComponent checks StorageService.getApiKey()
    → if null: render ApiKeySetupComponent
    → if set: render ReaderComponent

User enters API key
  → ApiKeySetupComponent calls StorageService.setApiKey(key)
  → AppState.apiKey.set(key)
  → ReaderComponent rendered

User pastes English text + clicks "Bắt đầu"
  → ReaderComponent calls TextProcessorService.splitToSentences(text)
  → AppState.sentences.set(result)
  → AppState.currentIndex.set(0)
  → SentenceCardComponent list re-renders with highlight on index 0

User types translation + presses Enter / clicks Submit
  → SentenceCardComponent calls GeminiService.evaluateTranslation(original, translation)
  → AppState.isLoading.set(true)
  → On success: AppState.currentFeedback.set(feedback)
  → On error: display error message
  → AppState.isLoading.set(false)

User clicks "Tiếp theo"
  → ReaderComponent increments AppState.currentIndex
  → SentenceCardComponent updates highlight
  → AppState.currentFeedback.set([]) (reset for new sentence)

User clicks "Gợi ý"
  → HintButtonComponent calls GeminiService.getKeywords(original)
  → Keywords displayed below input
```

**Key Decisions:**
- `ReaderComponent` owns toàn bộ app state qua Signals — single source of truth
- Services (`GeminiService`, `TextProcessorService`, `StorageService`) are stateless và pure — dễ test
- Không dùng NgRx hay external state library — Signals đủ cho app nhỏ này
- Gemini API được gọi trực tiếp từ browser — API key do user tự cung cấp nên không có vấn đề bảo mật expose key
